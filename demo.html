Styles

A style applies to a block of text. The MarkupEditor supports the standard HTML styles of P, H1, H2, H3, H4, H5, and H6. Styles cannot contain other styles. So, for example, you can't embed an H3 style in an H1 style header.

The toolbar refers to P style as "Normal". The notion of a style is not native to HTML (even if it is to CSS), but it's pretty well understood by every person who has ever used a word processor, spreadsheet, or even the most primitive text editing tool these days.

Lists

The MarkupEditor supports both numbered and bulleted style lists. The should behave like you would expect, maintaining the list style and progressing through the proper markers on sub-lists. You can switch styles using the toolbar and interchange styles within lists.

Indenting

Like the other styles, indenting applies to a block of text, although you can indent both normal paragraphs and headers. Once you indent, the toolbar enables the outdent button. You can indent multiple times.

Formats

Text can be formatted to present as bold, italic, underline, code, strikethrough, subscript, or superscript. Unlike styles, formats can be combined, so for example, a bold, italic, underlined section of text works just fine.

Inserted Elements

You can insert and edit links, images, and tables in the document. Each of these elements requires some kind of user input to create them. The MarkupEditor exposes an additional element-type-specific toolbar for each of these elements when you create them. To edit one of these elements, select an existing link, image, or table, and then expose the toolbar for that element.

Links

To insert a link, you need to start by selecting some text to link-to. When you've done that, the link button is enabled, and when you select it, the link toolbar appears below. Enter a URL and Save. Cancel just closes the toolbar, leaving the original text/element unchanged. You can leave the toolbar open as you select other elements in the text, which allows you to view and/or edit existing elements more easily. To delete an existing link, clear out the URL and press Save.

Images

Images have to be inserted at a point in some text, or at the beginning or end of some text. If you have text selected (i.e., one or more characters of text), the image button is disabled. When you have the cursor at a point in text, press the image button to expose the image toolbar. To insert an image in this version of the MarkupEditor, you need to specify a URL. You can (and should!) provide additional descriptive text to go along with it, which is the right thing to do for people with visual impairment. You can scale the image to make it smaller and fit better in your document.

Here is an inline image: . Select it and then press the image button to see its URL, alt text, and scale settings.

Again, you can Save your changes or Cancel, closing the toolbar. You can leave the toolbar open to view other image elements. To delete an image, you can clear out the URL and press Save.

Extending the MarkupEditor

The File toolbar is part of the demo, but not part of the core MarkupEditor. Check out the ContentView in SwiftUIDemo and ViewController in the UIKitDemo to see details of how it was done. The demo starts with some initial HTML taken from demo.html. You can open a different html file to see how it behaves. You can also toggle a view of the raw HTML off-and-on from the File toolbar. The raw HTML updates as you edit the document, to show what the MarkupEditor is doing behind the scenes.
