The file you're looking at is just a simple HTML file that includes all the kinds of elements supported by the MarkupEditor by default. This document also serves as a brief tutorial on using the MarkupEditor as an end-user, along with some information for developers.

Styles

Overall Document Styling

When you load an HTML document into the MarkupEditor, the document is styled using markup.css. In this demo, you can use the File toolbar above to examine the raw HTML of this demo.html document. You will see it is "clean" in the sense that it contains no embedded attributes like style, font, or sizing. The overall document styling is up to you as a developer (and it's my intent to have it easily customizable). When your end users see the word Style in the toolbar, it refers to paragraph style.

Paragraph Style

This is a Normal style paragraph. There is a title at the top at the top of this document that uses an H1 style. As you select in the text, the toolbar indicates the paragraph style of the element you selected.

A style applies to a block of text. The MarkupEditor supports the standard HTML styles of P, H1, H2, H3, H4, H5, and H6. Styles cannot contain other styles. So, for example, you can't embed an H3 style in an H1 style header.

The toolbar refers to P style as "Normal". The notion of a style is not native to HTML (even if it is to CSS), but it's pretty well understood by every person who has ever used a word processor, spreadsheet, or even the most primitive text editing tool these days.

Lists

The MarkupEditor supports both numbered and bulleted style lists. The should behave like you would expect, maintaining the list style and progressing through the proper markers on sub-lists. You can switch styles using the toolbar and change styles within lists.

Indenting

Like the other styles, indenting applies to a block of text, although you can indent both normal paragraphs and headers. Once you indent, the toolbar enables the outdent button. You can indent multiple times, and you can outdent until there is no more indenting to outdent.

Formats

Text can be formatted to present as bold, italic, underline, code, strikethrough, subscript, or superscript. Unlike styles, formats can be combined; so for example, a bold, italic, underlined section of text works just fine.

Inserted Elements

You can insert and edit links, images, and tables in the document. Each of these elements requires some kind of user input to create them. The MarkupEditor exposes an additional element-type-specific toolbar for each of these elements when you create or edit them. To edit one of these elements, select an existing link, image, or table, and then expose the toolbar for that element.

Links

To insert a link, you need to start by selecting some text to link-to. When you've done that, the link button is enabled, and when you select it, the link toolbar appears below. Enter a URL and Save. Cancel just closes the toolbar, leaving the original text/element unchanged. You can leave the toolbar open as you select other elements in the text, which allows you to view and/or edit existing elements more easily. To delete an existing link, clear out the URL and press Save.

Images

Images have to be inserted at a point in some text, or at the beginning or end of some text. If you have text selected (i.e., one or more characters of text), the insert image button is disabled. When you have the cursor at a point in text, press the insert image button to expose the image toolbar. To insert an image in this version of the MarkupEditor, you need to specify a URL. You can (and should!) provide additional descriptive text to go along with it, which is the right thing to do for people with visual impairment. You can scale the image to make it smaller and fit better in your document.

Here is an inline image: . Select it and then press the image button to see its URL, alt text, and scale settings.

Again, you can Save your changes or Cancel, closing the toolbar. You can leave the toolbar open to view other image elements. To delete an image, you can clear out the URL and press Save.

Tables

Tables have to be inserted at a point in some text or between paragraphs. As in the case for images, if you have text selected (i.e., one or more characters of text), the insert table button is disabled. When you have the cursor at a point in text, press the insert table button to expose the table toolbar. The create table button on the left side lets you create a table with the number of rows and columns you want. Once you have created the table, or you select a table in the document you're editing, the create table button is disabled. In other words, you cannot insert a table into a table.

The Add buttons in the table toolbar let you add a header at the top of the table or add rows/columns above/below the point you have selected in the table. The delete buttons remove the row/header you have selected in the table. To remove the header, select inside of it and press the remove row button.

Undo/Redo

The undo and redo buttons should do what you expect and are also accessible via the menu and hot-keys. There is currently a bug that may make the insertion caret disappear when you perform undo using a hot-key like Ctrl-Z. Right now only the formatting (bold, italic, etc) and paste operations are properly undo-able.

Extending the MarkupEditor

The File toolbar is part of the demo, but not part of the core MarkupEditor. The MarkupToolbar does not include the FileToolbar you see in the demo. Check out the ContentView in SwiftUIDemo and ViewController in the UIKitDemo to see details of how it was done. The demo starts with some initial HTML taken from demo.html. You can open a different html file to see how it behaves. You can also toggle a view of the raw HTML off-and-on from the File toolbar. The raw HTML updates as you edit the document, to show what the MarkupEditor is doing behind the scenes. Note that normally (i.e., when using the MarkupEditor outside of the demo), typing is a very lightweight operation. Because the demo is set up to update the raw HTML as you type, it's a bit more heavyweight in the demo.

The demo makes use of the leftToolbar option in the MarkupToolbar which makes it simple to add your own toolbar to the left (and/or with rightToolbar for the other side) of the standard MarkupToolbar. You can just as easily construct your own toolbar in SwiftUI using using the toolbar-level views like StyleToolbar and FormatToolbar, or with the LabeledToolbar and ToolbarButtons used within them.
